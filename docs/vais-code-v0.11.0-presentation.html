01

개요

VAIS Code는 무엇이고, 왜 필요한가?

VAIS Code란?

Claude Code 플러그인으로 동작하는 9단계 AI 개발 워크플로우 도구

🎯 문제

AI 코딩 도구에게 "만들어줘" 하면
기획 없이 바로 코딩 → 요구사항 누락,
설계 없는 코드, 검증 불가

💡 해결

조사→기획→설계→구현→검증까지
단계별 산출물을 강제하여
체계적 개발 워크플로우 보장

✨ 결과

기획서 기반 코딩 규칙 준수,
설계 대비 Gap 분석으로
90%+ 일치율 자동 검증

Claude Code Plugin MIT License 한국어 네이티브 v0.11.0

핵심 가치 & 차별점

기획서가 없으면 코딩 시작 불가.
기획서에 코딩 규칙, 기능 목록, 정책이 포함되어
이후 모든 단계의 단일 진실 소스(SSOT) 역할

일치율 90% 미만이면 최대 5회 자동 반복.

모델 비용 최적화

판단은 Opus, 구현은 Sonnet — 역할별 모델 분리

opus ━━━━━━━━━━━━━━━━━━━━ │ │ 판단, 전략, 아키텍처 │ ├── Manager (프로젝트 기억/판단) └── Tech Lead (아키텍처/오케스트레이션) sonnet ━━━━━━━━━━━━━━━━━━━━ │ │ 코드 구현, 문서 작성 │ ├── Designer (IA/와이어프레임/설계) ├── Frontend Dev (FE 구현) ├── Backend Dev (BE 구현 + DB) └── Reviewer (Gap/QA/리뷰)

핵심 원리

  • 기획/아키텍처: 정확성 우선 → opus
  • 코드 구현: 속도/비용 효율 → sonnet
  • 단순 탐색: haiku 활용 가능

비용 절감 효과

9단계 중 2단계만 opus 사용.
나머지 7단계는 sonnet으로 처리하여
전체 비용 60~70% 절감 (추정)

훅(Hook) 시스템

Claude Code 이벤트에 자동 반응하는 5개 훅

훅이벤트스크립트역할
SessionStart 세션 시작 hooks/session-start.js .vais/ 디렉토리 생성, 현재 피처 상태 표시
PreToolUse Bash 실행 전 scripts/bash-guard.js rm -rf, DROP TABLE, sudo 등 위험 명령 차단
PostToolUse Write|Edit 후 scripts/doc-tracker.js 문서 작성 감지 → 워크플로우 상태 자동 업데이트 + memory 기록
UserPromptSubmit 사용자 입력 scripts/prompt-handler.js 의도 감지 → 적절한 /vais 명령 제안, 체이닝/범위 파싱
Stop 응답 완료 scripts/stop-handler.js 진행률 프로그레스바 + 다음 단계 안내 출력

v0.11.0 변경

no-op 훅 2개 (notification.js, subagent-stop.js) 제거 → 불필요한 오버헤드 제거

Gap 분석 — 자동 검증 루프

설계 문서 vs 구현 코드를 자동 비교, 90% 미만이면 자동 재시도

프로세스

  1. 피처 레지스트리에서 요구사항 추출
  2. Glob/Grep으로 구현 코드 탐색
  3. 각 요구사항의 구현 여부 판정
  4. 일치율 산출: (구현 수 / 전체) × 100
  5. OWASP Top 10 보안 검사
  6. QA 시나리오 생성
  7. Gap 방향 판단: 코드 수정? 스펙 수정?
  8. 90% 미만 → 최대 5회 자동 반복
check 시작 │ ▼ 빌드 검증 (npm build / tsc) │ ▼ Gap 분석 실행 │ ├─ 일치율 ≥ 90% → 통과 ✅ │ └─ 일치율 < 90% │ ▼ Gap 방향 판단 ├─ 코드 누락 → 코드 수정 └─ 스펙 과잉 → 사용자 확인 │ ▼ 반복 (iteration++) │ ├─ iteration ≤ 5 → 재분석 🔄 └─ iteration > 5 → 중단, 보고 🛑

설정 (vais.config.json)

"gapAnalysis": {
  "matchThreshold": 90,
  "maxIterations": 5,
  "autoIterate": true
}

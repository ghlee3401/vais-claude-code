## 1. ONBOARDING.md 구조 spec

### 섹션 골격

```markdown
# VAIS Code — Onboarding (5분 읽기)

> 이 파일의 책임: 처음 본 AI/사람이 이 repo 구조를 5분 안에 파악하도록 돕는 진입 가이드.
> 더 깊이 파야 하면 → CLAUDE.md (Claude Code) / AGENTS.md (Cursor/Copilot) / SKILL.md (vais 명령어)

## What This Is (1분)

(프로젝트 한 단락 + v0.62.0 + 핵심 컨셉 3가지)

## Quick Start (1분)

(3개 진입 시나리오: vais 명령어 / 코드 읽기 / 새 피처 만들기)

## Architecture (1분, Mermaid)

(아래 §3 그래프)

## 진입점 역할 표 (1분)

(아래 §2 표)

## Next Steps (1분)

(워크플로우 1개 예시 — `/vais ceo ideation {idea}` → ... → commit)
```

### 분량 한계

160 줄 이내. 초과 시 가독성 손상 → 진단 후 분할 검토.

## 2. 진입점 역할 표 spec

```markdown
| 파일 | 대상 | 역할 | 언제 보나 | 길이 |
|------|------|------|----------|:----:|
| ONBOARDING.md | 모든 AI/사람 (처음) | 진입 가이드 — 5분 파악 | 처음 1번 | ~150줄 |
| CLAUDE.md | Claude Code | 프로젝트 지침 — Rules + Project Structure + Workflow | Claude Code 세션 시작 시 자동 로드 | ~400줄 |
| AGENTS.md | Cursor / Copilot / 일반 AI | Claude 외 AI 호환 지침 (CLAUDE.md 의 핵심 부분 추출) | 다른 AI 도구 사용 시 | ~200줄 |
| skills/vais/SKILL.md | Claude Code (skill 호출) | `/vais` 명령어 진입점 — phase + 액션 라우팅 | `/vais` 호출 시 자동 로드 | ~250줄 |
```

> 길이는 측정값 (각 파일 wc -l 기준). 진입 우선순위는 "처음 본 사람" → ONBOARDING → 의도에 따라 분기.

## 3. Mermaid 의존 그래프 spec

```mermaid
flowchart TB
    ONBOARD[ONBOARDING.md<br/>5분 진입] --> CLAUDE[CLAUDE.md<br/>Claude Code 지침]
    ONBOARD --> AGENTS[AGENTS.md<br/>Cursor/Copilot 호환]
    ONBOARD --> SKILL[skills/vais/SKILL.md<br/>/vais 명령어]

    SKILL --> PHASES[skills/vais/phases/<br/>ceo·cpo·cto·cso·cbo·coo·ideation]
    SKILL --> UTILS[skills/vais/utils/<br/>status·init·next·commit...]

    PHASES --> AGENTSDIR[agents/<br/>6 C-Level + 38 sub-agent]
    AGENTSDIR --> HOOKS[hooks/<br/>session-start·design-mcp-trigger·ideation-guard]
    AGENTSDIR --> SCRIPTS[scripts/<br/>doc-validator·auto-judge·patch-*]

    CLAUDE --> CONFIG[vais.config.json<br/>워크플로우·게이트·C-Suite 설정]
    PHASES --> CONFIG
    AGENTSDIR --> CONFIG

    AGENTSDIR --> DOCS[docs/{feature}/<br/>피처별 PDCA 산출물]
    DOCS --> DS[design-system/<br/>DS 카탈로그 박제 (mui v0.62.0+)]
```

> 부속: `lib/` (helpers), `templates/` (PDCA 템플릿), `output-styles/` (출력 스타일), `mcp/` (design-system server), `vendor/ui-ux-pro-max` (BM25 검색) 는 그래프 외 — ONBOARDING 본문에서 1줄씩 언급.

## 4. CLAUDE.md 정비 spec

### 추가/수정 항목

1. **상단에 "이 파일의 책임" 1줄 추가** (D-4 형식)
2. **"What This Project Is" 절 강화** — v0.62.0 명시 + design-system 추가
3. **진입 순서 안내 1단락** — "처음 본 AI 는 ONBOARDING.md 부터 → 본 파일은 Claude Code 전용 지침" 명시
4. **catalog.json 정체 1줄** — "Auto-generated by `scripts/build-catalog.js` — do not edit manually" Project Structure 섹션 또는 Key Configuration 절에 추가
5. **release-engineer 언급 제거** — Project Structure 절 내 sub-agent 목록 (line 37 부근) 에서 release-engineer 행 삭제, "38 sub-agent" 카운트 업데이트

### 제거하지 않는 항목

- "Mandatory Rules" 14개 모두 유지 (현재 유효)
- Agent Architecture 표 그대로
- File Conventions / Do NOT 그대로

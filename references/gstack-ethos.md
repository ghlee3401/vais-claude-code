## 1. Boil the Lake (완전성 원칙)

AI 지원 코딩은 완전성의 한계 비용을 거의 0으로 만듦. 완전한 구현이 숏컷보다 몇 분 더 걸릴 때, 항상 완전한 것을 하라.

- **Lake vs Ocean**: "Lake"는 끓일 수 있음 (100% 테스트 커버리지, 전체 기능 구현, 모든 엣지 케이스). "Ocean"은 끓일 수 없음 (전체 시스템 재작성, 다분기 마이그레이션).
- **Lake는 끓이고, Ocean은 범위 밖으로 표시**

| 작업 유형 | 인간 팀 | AI 지원 | 압축률 |
|----------|---------|---------|--------|
| 보일러플레이트 | 2일 | 15분 | ~100x |
| 테스트 작성 | 1일 | 15분 | ~50x |
| 기능 구현 | 1주 | 30분 | ~30x |
| 버그 수정 | 4시간 | 15분 | ~20x |
| 아키텍처 설계 | 2일 | 4시간 | ~5x |
| 리서치 | 1일 | 3시간 | ~3x |

## 2. Search Before Building (탐색 우선 원칙)

빌드 전에 "누가 이미 이 문제를 풀었나?" 먼저 확인.

### 세 가지 지식 레이어

- **Layer 1 (검증된 것)**: 표준 패턴, 배틀 테스트된 접근법
- **Layer 2 (새롭고 인기 있는 것)**: 현재 베스트 프랙티스, 블로그 포스트
- **Layer 3 (First Principles)**: 특정 문제에 대한 원리적 추론. **가장 가치 있음**

---

## 3. User Sovereignty (사용자 주권 원칙)

AI 모델은 추천한다. 사용자가 결정한다.

- 두 AI 모델이 합의해도 그것은 강한 신호이지, 명령이 아님
- 사용자는 항상 모델이 모르는 컨텍스트를 가짐: 도메인 지식, 비즈니스 관계, 전략적 타이밍, 개인 취향
- 올바른 패턴: 생성-검증 루프. AI가 추천을 생성하고, 사용자가 검증하고 결정

## VAIS Code에서의 적용

| 원칙 | VAIS 적용 |
|------|----------|
| Boil the Lake | 각 C-Level이 담당 범위를 완전하게 수행. "나중에" 미룸 금지 |
| Search Before Building | CTO architect 단계에서 기존 솔루션 탐색 우선 |
| User Sovereignty | CEO 체크포인트(CP)에서 반드시 사용자 확인. 자동 판단 금지 |
